## Product

**DaFit Canvas** ‚Äî a cross-platform desktop app for visually designing smartwatch faces compatible with the **Da Fit / MoYoung** ecosystem. Output of the editor is a `.zip` package that the external `dawft` CLI converts into a binary watch face uploaded via `dawfu`.

Watch hardware target:
- **240√ó240** screen, **16-bit RGB565**, little-endian, no alpha
- Type **C** binary (`fileType C`, `fileID 0x81`, `faceNumber 50001`)

## TypeScript gotchas

`tsconfig.app.json` sets `verbatimModuleSyntax: true`, `noUnusedLocals`, `noUnusedParameters`, `erasableSyntaxOnly`. Practical effects:
- Use `import type { Foo }` for type-only imports ‚Äî plain `import { Foo }` will fail the build when `Foo` is a type
- No enums or constructor-parameter properties (`erasableSyntaxOnly`) ‚Äî use unions/const objects instead
- Unused locals/params are errors, not warnings

`tsconfig.json` is a solution file; edit `tsconfig.app.json` for `src/` and `tsconfig.node.json` for build-tool configs.

## Domain: watch face format

Knowledge below is product-specific and not derivable from code ‚Äî keep it in mind when working on `export/`, `canvas/`, or element definitions.

### `watchface.txt` structure

Header (required):
```
fileType    C
fileID      0x81
dataCount   <count of faceData lines>
blobCount   <total BMPs>
faceNumber  50001
```

Then one `faceData` line per element:
```
faceData   <TYPE>   <INDEX>   <X>   <Y>   <W>   <H>   # comment
```
- `TYPE` ‚Äî hex element type (table below)
- `INDEX` ‚Äî starting blob index in the BMP sequence
- `X, Y` ‚Äî top-left position
- `W, H` ‚Äî width / height

A `blobCompression` section is auto-generated by `dawft` ‚Äî **the editor does not emit it**.

### Element TYPE ‚Üí blob count

Each element type consumes a fixed number of `dbNNN.bmp` files starting at `INDEX`:

| TYPE | Element | Blobs |
|------|---------|-------|
| 0x01 | Background | 1 |
| 0x00 | Background strips | 10 |
| 0x40 / 0x41 / 0x43 / 0x44 | Time H1 / H2 / M1 / M2 | 10 each |
| 0x30 | Day number | 10 |
| 0x11 | Month number | 10 |
| 0x60 | Day name | 7 |
| 0x10 | Month name | 12 |
| 0x70 / 0x80 / 0x90 | Steps / HR / Kcal progress | 11 each |
| 0x73 / 0x83 / 0x93 | Steps / HR / Kcal digits | 10 each |
| 0xd1 | Battery image | 10 |
| 0xf0 | Separator/static | 1 |
| 0x45 / 0x46 | AM / PM indicator | 1 each |

`blobCount` in the header must equal the sum of blobs across all `faceData` entries.

### BMP output

- Windows BMP, **16-bit RGB565, little-endian, no alpha**
- Filename pattern: `db000.bmp`, `db001.bmp`, ‚Ä¶ (zero-padded, sequential, matches `INDEX`)
- RGB565 encoding:
  ```ts
  const r = (pixel.r >> 3) & 0x1F
  const g = (pixel.g >> 2) & 0x3F
  const b = (pixel.b >> 3) & 0x1F
  const rgb565 = (r << 11) | (g << 5) | b
  ```

### Dynamic placeholders (preview-only in MVP)

Tokens that render as live values in the editor preview but are emitted as digit/icon blobs at export time: `{{HH}}`, `{{mm}}`, `{{DD}}`, `{{MM}}`, `{{BATTERY}}`, `{{STEPS}}`, `{{HEART_RATE}}`.

### Export pipeline

1. Render each element to its frame(s) on an offscreen 240√ó240 canvas
2. Encode each frame as RGB565 BMP ‚Üí `dbNNN.bmp`
3. Emit `watchface.txt` with header + one `faceData` per element
4. Zip everything ‚Üí user runs `dawft create folder=‚Ä¶ output.bin` then `dawfu upload address=XX:XX:XX:XX:XX:XX output.bin`

## Engineering rules

- Strong TypeScript typing ‚Äî no `any` in domain types (Element, Layer, Project)
- Modular by feature folder (`canvas/`, `export/`, etc.), not by file kind
- Reusable hooks for canvas interaction (drag, resize, select) and store access
- Avoid premature abstraction ‚Äî three similar lines beats a generic helper
